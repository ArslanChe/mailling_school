Инструкция по всем файлам проекта informer

1) папка vk_bot:
	а) BotSending.py:
		1) sender : отправка сообщение text пользователю с id = id
		2) send_stick : отправка стикера с номером number пользователю с id = id
		3) send_photo : отправка фотографии url пользователю с id = id
		4) db - подключение и работа с базой из файла wrapper_database
		5) Последовательность действий:
			•подключение к базе с использованием messanger = vk
			•взятие новое сообщение, если его нету exit()
			•статус 1 для сообщения - сообщение отправляется 
			•берем текст сообщение и id группы, которой его отправить, проверяем что сообщение не нулевое
			•по id группы берем id пользователей, кому отправить сообщение
			•проходим по id пользователей и отправляем сообщение, если не смогли обрабатываем ошибку
			•после отправки всех сообщений статус 2 для сообщения - сообщение отправлено
	б) BotTaken.py :
		1) create_keyboard : создание кнопки в диалоге с пользователем
		2) text_int : проверка, что отправили сообщение - число
		3) sender : отправка сообщение text пользователю с id = id
		4) send_stick : отправка стикера с номером number пользователю с id = id
		5) send_photo : отправка фотографии url пользователю с id = id
		6) db - подключение и работа с базой из файла wrapper_database
		7) Последовательность действий:
			•если пользователь регистрируется/отправляет число создается кнопка подтверждения прочтения
			•ожидание сообщения
			•берем id пользователя и текст который он отправил
			•если текст - число, то стараемся зарегистрировать пользователя и обнулить код подтверждения
			•если текст - строка, то проверяем это текст - подтверждения прочтения если да, то проверяем есть ли этот пользователь в базе
			•если пользователь найден, то обновляем подтверждение прочтения, если нет то пользователя нету или не зарегистрирован 
			•если текст не подтверждение прочтения, то ничего не делаем
	в) debag.py :
		1) Последовательность действий:
			•статус сообщения 0 - сообщение не прочитано
	г) wrapper_database :
		1) __init__ : подключение к базе данных
		2) get_message_id_from_new_messages : взятие id сообщения, которое нужно отправить
		3) update_messanger_status_in_new_messages : обновление статуса сообщения не отправлено/отправляется/отправлено
		4) get_text_and_group_id_from_message_information : взятие текста сообщения и id группы, которой надо отправить сообщение
		5) get_messenger_user_id_in_group : взятие id пользователей, которым надо отправить сообщение
		6) put_id_in_user_information : регистрация пользователя с id = id, по тексту подтверждения - code 
		7) debag_update : изменение статуса отправленных сообщений на не отправленные
		8) agree_send : подтверждение прочтения всех сообщений, отправленных пользователю с id = id
